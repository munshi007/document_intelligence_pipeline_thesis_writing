\section*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}

Industrial technical documents, such as product datasheets, operating manuals, and engineering specifications, represent one of the most information-dense and structurally complex document classes encountered in enterprise environments. Existing extraction approaches, ranging from commercial OCR engines to general-purpose Vision-Language Models (VLMs), systematically fail on these materials. Layout-only parsers destroy multi-column reading order and misinterpret embedded schematics as tabular data. Generative VLMs, when applied without structural constraints, produce confident but structurally invalid outputs due to token limits and hallucination when converting dense tables into structured JSON.

This thesis presents a multi-stage document intelligence pipeline specifically designed for industrial Visually Rich Documents (VRDs) to address these limitations. The approach decouples the document understanding problem into sequential layers. First, a DocLayout-YOLO detector isolates semantic regions, combined with deterministic column-aware reading-order sorting and a VLM-based referee for verification. Second, an algorithmic routing engine classifies each detected table region by computing vector line density and gap-ratio scores, dispatching grids to deterministic extractors or constrained generative specialists as appropriate. To enable local, low-latency deployment processing, the thesis introduces a "Dual-Student Architecture". Utilizing Low-Rank Adaptation (LoRA) and the Unsloth framework on a consumer GPU, visual reasoning tasks are distilled into an 11-Billion parameter Llama-Vision model, while logical data extraction capabilities are distilled into an independent 7-Billion parameter Qwen2.5-VL model.

Crucially, the pipeline introduces a "Two-Pass Markdown Harvest" architecture for data extraction. The first pass bypasses JSON token overhead limit by converting visual tables directly into high-density Markdown representations. The second pass applies a rigorous text-to-schema refinement, ensuring complete parameter recall and perfect Pydantic validation. The pipeline ultimately assembles a Hierarchical Knowledge Graph (HKG) from the extracted regions, encoding section breadcrumbs, spatial caption links, and cross-document references. Evaluated against rigorous structural and extraction benchmarks on dense industrial datasheets, the proposed solution recovers 100\% of technical parameters, maintaining absolute structural integrity while solving the truncation bottlenecks of traditional VLM single-pass extraction.
