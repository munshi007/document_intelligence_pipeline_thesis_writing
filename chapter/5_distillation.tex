\chapter{Knowledge Distillation and the Dual-Student Architecture}
\label{ch:distillation}

This chapter documents the application of Parameter-Efficient Fine-Tuning (PEFT) \citep{peft_houlsby} algorithms to enable specialized reasoning on localized hardware. Specifically, this research details the Knowledge Distillation \citep{knowledge_distill} sequence using Low-Rank Adaptation (LoRA) \citep{lora} and heavily optimized CUDA kernels via the Unsloth framework \citep{unsloth}. These techniques were used to transfer targeted analytical capabilities from a proprietary teacher model (GPT-4o \citep{gpt4v}) into the localized Edge pipeline.

A key design decision is that, rather than relying on a solitary monolithic model, this thesis introduces a \textbf{Dual-Student Architecture}. The intense cognitive load of document intelligence is split into two specialized tasks, governed by two independent models: an 11-Billion parameter \textit{Llama-Vision} student trained exclusively for Visual-to-Markdown optical mapping, and a 7-Billion parameter \textit{Qwen2-VL} student trained strictly for logical Markdown-to-JSON schema extraction.

\section{The Edge-Deployment VRAM Bottleneck}
\label{sec:vram_limits}

The absolute localized deployment target for this thesis pipeline is an NVIDIA GeForce RTX 3090 possessing 24 GB of VRAM. This strict hardware restriction prevents the concurrent execution of large, general-purpose VLMs. While 4-bit quantization (via \textit{bitsandbytes}) compresses weight parameters to allow baseline inference, conventionally training these models requires maintaining high-precision 32-bit optimizer states and large reverse-mode gradient graphs.

To train the dual visual and extraction specialists without exceeding the hard 24 GB VRAM limit, this pipeline employs a combination of LoRA constraints and Unsloth memory-mapped execution.

\section{Dual-Dataset Synthesis}
\label{sec:dataset_synthesis}

To train two distinct specialist models, two distinct Knowledge Distillation datasets were engineered. Both datasets used GPT-4o acting as the infallible "Teacher" to generate ground-truth output for roughly 1,000 dense industrial datasheets.

\subsection{Dataset 1: The Visual-to-Markdown Payload (For Llama-Vision)}
The first dataset targets the optical conversion limit. The teacher was provided high-resolution image crops of embedded engineering tables and schematics. The teacher was aggressively prompted to return pristine, structurally aligned Markdown grids that perfectly matched the visual tensor.
These image-to-text pairs were packaged for the Llama-Vision student, teaching the model to ignore generative hallucination entirely and simply act strictly as a high-fidelity semantic OCR engine.

\subsection{Dataset 2: The Markdown-to-JSON Schema Payload (For Qwen2-VL)}
The second dataset eliminates vision entirely, focusing purely on logical data extraction and formatting. The teacher was provided the intermediate Markdown crops generated in phase 1 alongside the rigid \texttt{LibrarianUniversalHardware} Pydantic schema constraints. The teacher populated the JSON keys accurately, categorizing electrical, mechanical, and connecting parameters.
These text-to-JSON pairs were compiled to train the Qwen2-VL extractor agent. By learning from pure text rather than raw images, the Qwen2-VL student considerably accelerates JSON mapping while adhering strictly to the Pydantic type requirements.

\section{Low-Rank Adaptation Framework (LoRA)}
\label{sec:lora_math}

To train the student models within the localized environment, Low-Rank Adaptation (LoRA) was implemented. Rather than explicitly altering the $d \times d$ pre-trained weight matrix $W_0 \in \mathbb{R}^{d \times d}$ during gradient descent, LoRA mathematically freezes $W_0$. 

The algorithm introduces two trainable decomposition matrices $A$ and $B$, possessing a constrained rank dimensionality $r$ where $r \ll d$. The forward pass incorporates the low-rank update $\Delta W$:
\begin{equation}
h = W_0 x + \Delta W x = W_0 x + BA x
\end{equation}

By restricting the rank to $r = 16$, the number of trainable parameters for both the Llama-Vision and Qwen2-VL runs was reduced by over 98\%, significantly lowering the VRAM required for gradient storage while maintaining the models' ability to conform to their specific training tasks.

\section{Optimization Engine: Unsloth and Flash Attention 2}
\label{sec:unsloth_training}

Mathematical parameter reduction via LoRA solves part of the VRAM bottleneck, but natively executing attention gradients across extended document contexts rapidly triggers Out-Of-Memory (OOM) failures. In standard Transformer architectures, the self-attention mechanism requires computing a sequence-to-sequence similarity matrix. For a sequence of length $N$, computing the softmax attention weights $S = \text{softmax}(QK^T / \sqrt{d})$ necessitates materializing an $N \times N$ matrix directly into High-Bandwidth Memory (HBM). When an industrial document necessitates a context window of $N=8192$ tokens, this single operation immediately demands several gigabytes of VRAM merely to store intermediate activations.

To solve these severe hardware limits, the entire distillation suite was executed utilizing the \textbf{Unsloth} \citep{unsloth} framework, which integrates the \textbf{Flash Attention 2} \citep{flash_attention} algorithm.

\subsection{Algorithmic Tiling and Flash Attention}
Flash Attention fundamentally restructures the GPU memory hierarchy during the forward and backward passes. Instead of loading the entire $Q$, $K$, and $V$ matrices from slow HBM into the ultra-fast (but highly constrained) SRAM, Flash Attention employs a technique known as \textit{Tiling}. It splits the matrices into smaller blocks and iteratively loads them into SRAM. 

Crucially, Flash Attention 2 optimizes the backward pass (gradient computation). Standard mathematically exact attention requires saving the large $N \times N$ matrix computed during the forward pass to apply the chain rule during backpropagation. Flash Attention 2 mathematically avoids this by re-computing the forward statistics (specifically the log-sum-exp stabilization variables) dynamically block-by-block on the backward pass. This algorithmic breakthrough means that VRAM usage scales linearly $O(N)$ with sequence length, rather than quadratically $O(N^2)$. For the Llama-Vision model processing visually dense 1024x1024 tensor grids alongside textual prompts, this optimization prevented instant CUDA memory failures.

\subsection{Memory-Mapped Triton Compilation}
Beyond Flash Attention, Unsloth provides radical memory optimizations specific to LoRA adapters. Standard implementation via the HuggingFace PEFT \citep{peft_houlsby} library initializes adapters using naive PyTorch operations. This frequently causes GPU fragmentation where the adapter tensors and the frozen base weights misalign in memory.

Unsloth bypasses standard PyTorch abstractions entirely. It compiles bespoke Triton \citep{triton} GPU kernels specifically designed for the specific base architectures (e.g., Llama and Qwen2). These kernels aggressively fuse operations; for example, the Rotary Position Embedding (RoPE) application, the SwiGLU activation function, and the Cross-Entropy Loss computation are executed within a single kernel launch. This prevents the GPU from writing large intermediate floating-point tensors back to HBM after every micro-operation.

With Unsloth integrated and executing atop 4-bit quantized base weights, training was executed for 1,000 steps with a global batch size of 8 (simulated via gradient accumulation). The process required merely 15 GB of VRAM at peak usage, taking roughly 4 hours per model on the RTX 3090. This makes enterprise-grade multimodal pipeline deployment accessible on consumer hardware, running within the strict constraints native to highly-secured edge facilities.

\section{Training Loss and Convergence Analysis}
\label{sec:training_loss}

Splitting the cognitive load across two distinct datasets prevented early memorization. Rather than struggling to learn both spatial mapping and JSON formatting simultaneously, the models isolated their respective tasks rapidly.

Figure \ref{fig:dual_loss_curve} illustrates the Cross-Entropy training loss trajectories recorded for both the Llama-Vision (Visual-to-Markdown) and Qwen2-VL (Markdown-to-JSON) models over 1,000 steps. 

When fine-tuning large pre-trained language models via LoRA, the learning rate dictates the step size of the optimizer as it updates the low-rank matrices. Selecting a static learning rate often results in sub-optimal convergence: if set too high, early gradients can destructively override pre-existing foundational weights, whereas setting it too low severely prolongs training time. To resolve this, dynamic scheduling is employed. 

As visualized in Figure \ref{fig:learning_rate_curves}, both student models were trained using a cosine decay schedule prefixed by a linear warmup phase. During the initial warmup phase, the learning rate is progressively scaled from zero to its maximum defined threshold. This prevents early, high-variance loss signals from inducing parameter divergence. Once the maximum threshold is reached, the cosine decay gradually tapers the learning rate along a trigonometric curve. This deceleration ensures that as the model approaches the theoretical loss minimum, the gradient step sizes become infinitesimally small, allowing the optimizer to settle smoothly into the optimal parameter configuration without overshooting.
