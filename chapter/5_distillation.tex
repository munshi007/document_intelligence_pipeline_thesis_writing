\chapter{Custom Model Distillation and PEFT Optimization}
\label{ch:distillation}

The multi-modal ingestion methodology detailed in preceding chapters fundamentally relies on localized Visual-Language Specialists for algorithmic routing verification and complex layout parsing. However, deploying state-of-the-art Generative Models (e.g., GPT-4V, Llama-Vision 70B) natively imposes extreme computational constraints that functionally prohibit edge-deployment across standard enterprise hardware.

This chapter documents the application of Parameter-Efficient Fine-Tuning (PEFT) algorithms to circumvent VRAM exhaustion. Specifically, this research details the Knowledge Distillation sequence utilizing Low-Rank Adaptation (LoRA) and heavily optimized CUDA kernels (Unsloth) to synthesize high-level spatial reasoning capabilities into an 11-Billion parameter framework compatible with a singular consumer Graphics Processing Unit (GPU).

\section{The Edge-Deployment VRAM Bottleneck}
\label{sec:vram_limits}

In deep learning, the Random Access Memory (RAM) required to train or execute a transformer scales quadratically with both context length and parameter count. The full-precision float32 tensor weights of a standard 70B parameter Vision-Language Model exceed 280 Gigabytes of VRAM merely to load the architectural backbone, mandating massive, multi-card data-center server racks (e.g., $8\times$ NVIDIA A100 instances).

However, the localized deployment target for this thesis pipeline is constrained to a singular NVIDIA GeForce RTX 3090 possessing just 24 GB of VRAM. This aggressive hardware restriction mandates profound memory optimization strategies. The baseline inference necessitates mathematical quantization, converting float32 bit-depths into 4-bit (int4) precision matrices via techniques such as BitsAndBytes.

While 4-bit quantization solves execution loads, \textit{training} a model dynamically requires generating heavy gradient checkpoint tensors. To effectively specialize an 11B parameter visual model (such as LLaMA-Vision) on custom technical layouts without triggering Out-Of-Memory (OOM) fatal errors, the standard backward propagation algorithm must be bypassed entirely.

\section{Low-Rank Adaptation (LoRA)}
\label{sec:lora_math}

To train a robust visual referee exclusively on 24 GB of VRAM, this thesis implements Low-Rank Adaptation (LoRA). Rather than explicitly altering the $d \times d$ pre-trained weight matrix $W_0 \in \mathbb{R}^{d \times d}$ during gradient descent, LoRA mathematically freezes $W_0$ entirely. 

The algorithm then introduces two drastically compressed, trainable spatial decomposition matrices $A$ and $B$, possessing a constrained rank dimensionality $r$ where $r \ll d$. The forward continuous-time propagation alters the weight accumulation $h$:

\begin{equation}
h = W_0 x + \Delta W x = W_0 x + BA x
\end{equation}

where $B \in \mathbb{R}^{d \times r}$ and $A \in \mathbb{R}^{r \times d}$. By severely restricting the rank integer (e.g., $r = 16$ or $r = 32$), the volume of trainable parameters diminishes by nearly 98\%, completely eviscerating the gradient footprint required inside the VRAM while retaining immense inferential flexibility.

\section{Distillation and Cross-Model Dataset Generation}
\label{sec:dataset_prep}

To utilize the frozen visual backbone for structured XML grid extraction (as documented in Chapter 4), a high-quality spatial dataset must be formulated. In standard Knowledge Distillation, the superior outputs of an immensely massive Teacher model (e.g., GPT-4V) are systematically curated and fed sequentially to the smaller Student model (Llama-Vision 11B).

A robust dataset of 1,000 industrial engineering datasheets consisting of diverse Table matrices, Schematic diagrams, and explicit 2-column key-values was generated. For each layout cropped via RT-DETR, the Teacher model processed the tensor and generated the desired deterministic XML `<ROW>` hierarchy.

The corresponding input-output pairs were formatted explicitly reflecting standard Llama multimodal training structures:
\begin{verbatim}
[
  {
    "role": "user",
    "content": [
      {"type": "image"},
      {"type": "text", "text": "Extract XML Grid <ROW><P><C><V>"}
    ]
  },
  {
    "role": "assistant",
    "content": [
      {"type": "text", "text": "<ROW><P>Voltage</P><C>Max</C><V>24 V</V></ROW>"}
    ]
  }
]
\end{verbatim}

\section{Fine-Tuning Execution (Unsloth Dynamics)}
\label{sec:unsloth_training}

To push memory optimization to its absolute absolute boundaries, the training script was compiled utilizing the \texttt{Unsloth} python libraries. Unsloth bypasses standard PyTorch computational graphs, replacing them with custom-coded, memory-mapped Triton/CUDA kernels mathematically optimized to compute identical LoRA derivatives simultaneously conserving 60\% more VRAM.

Figure \ref{fig:training_loss} plots the empirical trajectory logged sequentially by the Weights \& Biases (W\&B) tracking suite. The metric demonstrates pristine optimization. The student framework mathematically converged rapidly within the first 1,000 algorithmic steps without destructive over-fitting or catastrophic forgetting against the core LLaMA-Vision baseline.
